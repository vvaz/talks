CSS - Cascading Style Sheets

As nossas folhas de estilo são muito poderosas e podemos mudar como se comportam elementos, as suas cores, tamanhos, ordem, posição, a sua visibilidade, animações, etc..

Tal como o HTML, não é uma linguagem de programação, apenas podemos descrever regras de como formatar o nosso documento (DOM)

ex: Todos os h1 deverão ser vermelhos

							color: red;
						 }

não se esqueçam de acabar cada declaração com um ;

  • No elemento head com a tag style

Cores

Para percebermos o uso das cores num ecrã, temos de esquecer o que aprendemos em criança - Juntando mais cores, mais escura a cor vai ficar.

Em ecrãns, trabalhamos com luz, o que significa que quando todas as cores estão juntas, é produzido o branco.

Cores

O CSS compreende vários formatos de cores:

  • Com o nome em inglês de uma cor
  • Com o valor em HEX (Hexadécimal)
  • Com o valor em rgb / rgba
  • Com o valor em HSL (Hue, Saturation and Lightness)

Os nomes possíveis das cores podem ser consultados aqui: MDN Color Values

As declarações de CSS dos pais, vão afetar os seus filhos

Elementos Block

  • Respeitam as margens todas
  • força uma nova linha
  • Fica com a largura máxima se não tivermos definido

Elementos inline

  • Respeitam as margins e paddings à esquerda e direita mas não no top e bottom
  • Não podem ter width e height definidos
  • Deixam existir elementos à esquerda e direita

Elementos Inline-block

  • Outros elementos podem existir à esquerda e direita
  • Respeitam margens top e bottom
  • Respeitam height e width

Transições

Sintaxe:

					 		
					.elemento {
						transition: [property] [duration] [effect] [delay];
					}

Sintaxe:

					 		
					.elemento {
						transition: opacity 300ms ease-in-out 1s;
					}
[property]

[effect]

  • ease - transição que começa devagar, depois rápido e acaba devagar
  • linear - velocidade igual no inicio e no fim
  • ease-in - velocidade mais rápida no fim
  • ease-out - velocidade mais rápida no início
  • ease-in-out - começo e fim mais lento
  • cubic-bezier(n,n,n,n) - podemos definir a velocidade desejada

Propriedade Animation e Keyframes
Keyframes

Sintaxe:

					 		
						@keyframes [name] {
							from {
								[styles]
							}
							to {
								[styles]
							}
						}

Sintaxe:

					 		
						@keyframes [name] {
							from {
								height: 200px;
								background: orange;
							}
							to {
								height: 400px;
								background: blue;
							}
						}

Sintaxe:

					 		
						.element {
							animation: [name] [duration] [timing-function] [delay] [iteration-count] [direction] [fill-mode] [play-state];
						}
Os Keyframes representam o que a animação faz, e a "animation" representa como é que deve ser feito. 
					 		
						.element {
							animation: timeline 2s ease-in-out 0s infinite normal forwards paused;
						}
