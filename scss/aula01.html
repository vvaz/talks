SASS

SASS representa Syntactically Awesome Style Sheets, e é um dos preprocessadores CSS mais populares. Estende o CSS adicionando características de linguagens de programação tradicionais, particularmente linguagens de programação orientadas a objetos.

1. Encapsulation

Com o CSS estamos a trabalhar num espaço "Global". Não temos forma de nos proteger enquanto criamos as nossas declarações de CSS.

2. Variables

Se quisermos dar suporte à maior parte dos browsers não podemos usar variáveis.

3. Not composable

É muito dificil / às vezes impossível, ao estilizar um componente não reutilizar declarações

4. Bad modularity primitives

O CSS não foi desenhado para ser usado nas aplicações complexas que temos neste momento. Manter as coisas organizadas é um desafio.

Preprocessors

Define conceitos similares a CSS, mas o que estamos a escrever, não é CSS. Ajuda-nos a organizar ficheiros, adicionar variáveis e reutilizar código.

O Preprocessor vai pegar no código que escrevermos em Sass e compilar para CSS

É modular

Existe o Less, o Sass e o Stylus

Funcionalidades do SASS

Compatibilidade com CSS

É extremamente compatível com CSS, o que significa que qualquer CSS válido é também válido em SASS. Isto é muito útil porque significa que podemos começar a usar SASS num projeto existente sem ter de reescrever todo o CSS.

Funcionalidades múltiplas

SASS tem muitas funcionalidades que o CSS não tem, como variáveis, nesting, mixins, funções, e muito mais. Isto torna o SASS muito mais poderoso do que o CSS.

Preprocessors

  • Adiciona coisas que deveriam ser base do CSS
  • Os estilos são mais fáceis de escrever e mais fáceis de gerir
  • É muito mais fácil de manter os estilos organizados
  • É fácil de instalar
  • Existem algumas ferramentas muito boas para gerir SCSS

SASS consiste em duas sintaxes. A mais antiga é indent based e a mais recente é SCSS (Sassy CSS)

Syntax

style.sass - é indent based - não vamos usar

              .child-class
                color: red
            .second-class

Syntax

style.scss

          
            .quadrado {
              width: 100px;
              height: 100px;
              background: blue;
            }

Importing and Variables

Usar o @import em CSS resulta em mais um HTTP request, o que afeta a performance do site.

No Sass, o @import faz uma coisa diferente (e muito mais útil)

 
            
              @import '_variables'; // para importar um ficheiro
              @import 'layout/colors' // para importar um ficheiro dentro de uma pasta layout

Com o import, podemos importar outros ficheiros .scss, ficheiros .css, ou até libraries externas (como o fontawesome)

src/sass

 
          
            _layout.scss
            _variables.scss
            _typography.scss
            layout/colors.scss
            app.scss
            other.scss

Os ficheiros com o underscore inicial chamam-se de partials.

São desenhados para serem importados para o ficheiro principal e não vão gerar ficheiros CSS únicos

O preprocessor não vai ler ficheiros começados com "_"

Variáveis

Podemos guardar valores em váriaveis com o Sass e utilizar em todos os ficheiros .scss

 
            
              $error_color: #f00 !default;

              .alert-color {
                $text_color: #ddd;
                background-color: $error_color;
                color: $text_color;
                text-shadow: 0 0 2px darken($text_color, 40%);
              }

o !default quer dizer "unless set elsewhere" - ou seja, usa este valor, a menos que seja definido noutro sitio

o $text_color - está a definir uma variável local, só vai ser usado nesse block

O darken() é uma função do Sass que requer um valor em percentagem para escurecer uma certa cor.

Variávels

  • É possível usar expressões matemáticas simples.
  • Sass pode converter unidades
  • Todos os valores de CSS podem ser usados como variáveis

Variávels - Types

  • Números;
  • Cores;
  • Strings;
  • Booleans (True or False)
  • Lists

Comentários:

 
            
              /*
               * Este comentário vai manter-se no ficheiro compilado
               */
               .foo {
                  color: red; // este comentário vai ser apagado
               }
