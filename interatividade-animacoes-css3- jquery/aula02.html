Transições e Animações

Permite-nos:

  • Dar vida a elementos
  • Dar a percepção que algo tem movimento

Existem duas formas de gerar movimentos em CSS:

  • Propriedade Transition
  • Propriedade Animation + Keyframes

Transitions vs Animations

As transições de CSS, permitem mudar os valores das "properties" mas precisa sempre de existir um gatilho ou "trigger" (como o click ou o hover).

As mudanças das "properties" não têm efeito imediato. As alterações às properties vão ser aplicadas durante um periodo de tempo que se tem de definir.

As transições apenas têm um estado inicial e final. Não podemos definir pontos intermédios e o browser assume estes pontos por nós.

Ao contrário das transições, as animações permitem adicionar pontos intermédios aos elementos de HTML. As animações consistem em 2 passos, a animação definida na folha de estilos e um conjunto de "keyframes" que indicam o inicio, os pontos intermédios (se existirem) e o fim da animação.

Transições

Sintaxe:

             
        .elemento {
          transition: [property] [duration] [effect] [delay];
        }
            
          
          
           transition-property: color;
           transition-duration: 1s;
           transition-timing-function: linear;
           transition-delay: 0.5s;
          
          
          
           transition: color 1s linear 0.5s;
transition-property é usada para determinar que propriedades vamos animar 
		 		
					transition-property: color;

	 			
		 		
		 		
		 		transition-property: all;

translation-duration define quanto tempo a transição irá durar

Podemos defenir o valor em segundos ou milisegundos 

		 		
		 		/* em segundos */
		 		transition-duration: 1s;
		 		transition-duration: 0.5s;

		 		/* em milisegundos */
		 		transition-duration: 1000ms;
		 		transition-duration: 500ms;

transition-timing-funtion: usada para definir a curva de aceleração relativa à duração da animação-

		 		
		 		transition-timing-function: ease;
		 		transition-timing-funtion: ease-in;

transition-delay: usada para colocar um delay à animação, definido em segundos ou milisegundos-

		 		
		 		transition-delay: 0.5s

Se não for definido, não existirá delay

Sintaxe (shothand) para uma property:

             
        .elemento {
          transition: opacity 300ms ease-in-out 1s;
        }

ou todas as properties:

             
        .elemento {
          transition: all 300ms ease-in-out 1s;
        }

[property]

São as propriedades de CSS que podem ser animadas. (Nem todas as propriedades podem ser animadas)

Propriedades que podem ser animadas:

  • font-size
  • background-color
  • width
  • left
  • ...
  • https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_animated_properties
Propriedades que não podem ser animadas:
  • display
  • font-family
  • position
Felizmente existem sites que nos facilitam a vida - http://canianimate.com/

As propriedades melhores para animar

  • posição
  • scale
  • rotation
  • opacity

[effect]

  • ease - transição que começa devagar, depois rápido e acaba devagar
  • linear - velocidade igual no inicio e no fim
  • ease-in - velocidade mais rápida no fim
  • ease-out - velocidade mais rápida no início
  • ease-in-out - começo e fim mais lento
  • cubic-bezier(n,n,n,n) - podemos definir a velocidade desejada

Como podemos começar uma animação?

  • Com as pseudo class (Hover, focus, etc)
  • Mudanças de classes

Animações
Propriedade Animation e Keyframes
Keyframes

Sintaxe:

             
          @keyframes [name] {
            from {
              [styles]
            }
            to {
              [styles]
            }
          }

Sintaxe:

             
          @keyframes animacao {
            from {
              height: 200px;
              background: orange;
            }
            to {
              height: 400px;
              background: blue;
            }
          }

Sintaxe:

             
          .element {
            animation: [name] [duration] [timing-function] [delay] [iteration-count] [direction] [fill-mode] [play-state];
          }
Os Keyframes representam o que a animação faz, e a "animation" representa como é que deve ser feito. 
             
          .element {
            animation: timeline 2s ease-in-out 0s infinite normal forwards paused;
          }
